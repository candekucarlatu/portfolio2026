# This is NOT the Next.js you know

This version (Next.js 16) has breaking changes — APIs, conventions, and file structure may all differ from your training data. Read the relevant guide in `node_modules/next/dist/docs/` before writing any code. Heed deprecation notices.

Key things to remember for this codebase:

- **`params` is always a Promise.** `export default async function Page({ params }: PageProps<'/[lang]'>)` then `const { lang } = await params`.
- **Use the typed helpers `PageProps<'/[lang]'>`, `LayoutProps<'/[lang]'>`** generated by `next typegen` instead of declaring inline param types.
- **`proxy.ts` replaces `middleware.ts`.** Locale detection lives in `src/proxy.ts`.
- **Parallel route slots need `default.tsx`.** `app/[lang]/@drawer/default.tsx` returns `null`; `app/[lang]/default.tsx` re-exports the home so children stay rendered when the drawer is open.
- **Turbopack is the default** for both `dev` and `build`. There is no `--turbopack` flag.

# Candela Kucar Latu — Portfolio

A pannable bulletin-board portfolio built with Next.js 16, Tailwind v4, Framer Motion, and a JSON content layer. The home is a single 2500×1800 design-coordinate canvas that the user drags to explore. Each project opens as a Stripe-style modal on desktop and as a bottom sheet on mobile.

> **Audience for this file:** AI coding agents (Claude Code, Codex, Cursor, etc.) and human contributors. The most common task is **editing project content** — see [Editing project content](#editing-project-content).

## Editing project content

There are exactly **4 projects** at `content/projects/<slug>/`:

```
content/projects/
  scribd/{es,en}.json
  slideshare/{es,en}.json
  kaplan/{es,en}.json
  tacobell/{es,en}.json
```

Each file matches the `Project` Zod schema in `src/lib/content/schema.ts`. The shape is:

```ts
{
  slug: string,                  // must match the folder name
  card: { title, subtitle, color },   // shown as the sticky note on the home
  sections: Section[]            // rendered top-to-bottom in the drawer / standalone page
}
```

`color` ∈ `'green' | 'blue' | 'orange' | 'purple'` (sticky note background).

### Section types

The `sections` array uses a **discriminated union** keyed on `type`. Add or reorder sections freely.

| `type`         | Renders                                 | Required fields                                                                                    |
| -------------- | --------------------------------------- | -------------------------------------------------------------------------------------------------- |
| `hero`         | Title + subtitle + meta line            | `title`, `subtitle`, `meta: { duration, team, role }`                                              |
| `section`      | Eyebrow label + H2 + body paragraph     | `label`, `title`, `body`                                                                           |
| `stat-cards`   | 1–4 sticky-note cards in a row          | `cards: [{ value, label, color }]`                                                                 |
| `highlight`    | Bordered callout paragraph              | `body`                                                                                             |
| `image`        | Single image (3 widths)                 | `image: { src, alt, width?, height? }`, optional `caption`, `width: 'default' \| 'wide' \| 'full'` |
| `insight`      | Side-by-side text + image               | `title`, `body`, `image`, optional `reverse: true`                                                 |
| `next-project` | Closing card linking to next case study | `slug`, `title`, optional `image`                                                                  |

**Body text supports plain strings only** — no Markdown rendering. Keep paragraphs as single sentences for now; add a Markdown section type if richer formatting becomes a recurring need.

### Adding or editing a section — recipe for an LLM

1. Open `content/projects/<slug>/<locale>.json`.
2. Find the `sections` array.
3. Insert/edit a section object that matches one of the discriminated-union variants in [Section types](#section-types).
4. Use the **same `slug`** in both `<locale>` files. Both must list the **same sections in the same order**, with translated text.
5. Save. Dev server hot-reloads. Validation errors surface as 404s — check the dev server log for a `[content] invalid project …` message.

**Image paths** must start with `/canvas/projects/<slug>/` and the file must exist in `public/canvas/projects/<slug>/`. Prefer `.jpg` for photos and `.png` for UI screenshots. Always include `alt` text in the locale's language.

### Adding a new project

1. Pick a slug like `coolproject`.
2. Create `content/projects/coolproject/es.json` and `en.json`.
3. Register the slug in `src/lib/content/loader.ts` (the `projectLoaders` map). The slug union is derived from this map — TypeScript will fail to compile until you add it.
4. Add the project to `src/components/canvas/itemPositions.ts` with its `photo` and `note` rectangles in design coordinates (relative to the 2500×1800 board).
5. Drop a transparent PNG of the project's photo group into `public/canvas/groups/<slug>.png`.

### Adding a new section type

1. Define a new Zod variant in `src/lib/content/schema.ts` and add it to the `Section` discriminated union.
2. Create the renderer at `src/components/case-study/sections/<Name>.tsx`.
3. Wire it into the `switch (section.type)` in `src/components/case-study/CaseStudy.tsx`.

### Adding strings to the UI dictionary

Edit `src/lib/i18n/dictionaries/{es,en}.json`. Keep keys identical across locales — the `Dictionary` type is inferred from `es.json`.

## The pannable canvas

The home is a single `<motion.div>` of fixed size **2500 × 1800** (design coordinates) that the user drags. All items are absolutely positioned within it.

- Items live in `src/components/canvas/itemPositions.ts` with `{ x, y, w, h }` measured from the top-left of the board.
- The canvas centers on the **About Me** rectangle on first load (`getCenteredOffset` in `PortfolioCanvas.tsx`).
- Drag is bounded so the board never reveals empty space (`getDragConstraints`).
- `prefers-reduced-motion` disables drag entirely (the board renders centered and static).

To change which layer is centered on load, edit `getCenteredOffset` in `src/components/canvas/PortfolioCanvas.tsx` to use a different rectangle from `itemPositions.ts`.

## i18n

- Default locale: **`es`**. Other: **`en`**.
- `src/proxy.ts` reads `Accept-Language` and redirects `/` → `/<locale>`.
- All routes live under `app/[lang]/`. The `lang` param is validated via `isLocale()` from `src/lib/i18n/config.ts`.
- The language switch (top-right button) swaps the first segment of the current pathname.

## Things NOT to touch without asking

- `public/canvas/groups/*.png` and `public/canvas/bg-pegboard-tile.png` — exported pixel-perfect from the Figma. Re-exporting requires the figma-console MCP and the Portfolio Figma file open.
- The note frame coordinates in `src/components/canvas/itemPositions.ts` — these come from the Figma file (each note has a tilt and exact position). If they change, the sticky-note overlays drift away from the photo composition underneath.
- Locale segment shape (`/[lang]/work/[slug]`) — both the proxy and the parallel/intercepted route conventions assume this exact path structure.
